Section 01 — Honest Audit
9 Issues That Will Break the Current Plan
The existing files are well-intentioned but contain technical flaws that will cause the SEO to fail silently — meaning you'd deploy everything and see no ranking improvement.
🔴 Critical — Will silently kill rankings
Critical Bug #1
SEOMeta uses client-side JS — SSG won't work
The SEOMeta component injects meta tags via useEffect — meaning meta only appears after JavaScript runs. With vite-ssg, Google crawls the pre-rendered HTML before JS executes. The static HTML will have blank titles and descriptions. Google will see: <title>Vite App</title> — not your custom title. Fix: Replace with react-helmet-async + vite-ssg's head injection pipeline.
Critical Bug #2
Fake AggregateRating schema — Google penalty risk
The buildSoftwareAppSchema() returns "ratingValue": "4.9", "reviewCount": "128" — hardcoded fabricated numbers. Google's manual review team actively hunts fake review schema and can issue a manual penalty that removes the site from Google entirely for months. Fix: Remove AggregateRating entirely until you have real, verifiable reviews. Replace with genuine EducationalApplication schema.
Critical Bug #3
routes.tsx has a top-level await syntax error
Lines like const { LearnHubPage } = await import(...) at the module's top level will throw SyntaxError: await is only valid in async functions unless the entire module is treated as async (which route files typically aren't). This will crash the entire build. Fix: Convert all imports to standard lazy imports with React.lazy().
Critical Bug #4
ActivityPage and AppModePage are empty stubs
These components render <div id="app-mode-freePaint" /> — an empty div with no content. When vite-ssg pre-renders these pages, Google gets a blank page with a title and description but no body content. This is "doorway page" territory and can result in Google deindexing those pages. Fix: Every page needs actual readable HTML content.
🟡 High — Will cap rankings significantly
High Priority #5
No per-page OG images — social CTR will be terrible
Every one of the 52 pages uses the same og-default.jpg. When parents share /trace-a on Facebook, it shows a generic thumbnail — not a letter A preview. Click-through from social will be 60–80% lower than it should be. Fix: Use Vercel's @vercel/og edge function to auto-generate per-page social images — no manual work needed.
High Priority #6
Content is too thin on revenue-driving pages
Google's Helpful Content system (2023–2024 updates) demotes pages with thin, AI-sounding content. The education pages (/for-parents, /for-teachers) need genuine 1,200–2,000 word articles. The learn pillar pages are defined in config but have no actual article content in the code. Fix: Write proper articles — not just FAQs.
High Priority #7
Missed programmatic expansion: numbers + shapes
The plan covers A–Z (26 pages) but ignores equally high-volume targets: trace numbers 0–9 ("trace number 3 for kids" = ~3,200/mo), and shape tracing (circle, triangle, square, star, heart = ~20K combined searches/mo). That's 37 more easy programmatic pages the template could generate instantly. Fix: Extend LetterTracePage template to a universal TracePage component.
High Priority #8
No backlink acquisition strategy that runs automatically
The plan says "outreach to parenting bloggers" — manual work that will never happen. Automated backlink generation requires: an embeddable widget (teachers drop an iframe → school domain backlink), a press kit page (journalists link it), and a free downloadable resource page (.edu sites link naturally). Fix: Build these three pages and links come in passively.
🟢 Medium — Significant missed opportunity
Medium Priority #9
YouTube is the #2 search engine — completely ignored
Parents search YouTube first for "how to trace letters kids." A 60-second screen recording of a child using Draw in the Air, uploaded to YouTube as a demo video, and embedded on the relevant page, creates: a YouTube backlink, YouTube ranking signals, and an additional discovery channel. Each activity page needs a demo video. Fix: Record 4–6 simple screen recordings and embed them on activity pages.
Medium Priority (Bonus)
Chrome Web Store listing — free 50K/mo channel
Draw in the Air is a PWA (it has a manifest.json). Publishing it to the Chrome Web Store costs $5 one-time. School Chromebooks use Chrome Web Store as their primary app discovery channel. "Educational games Chrome" gets 8,000+ searches/month from teachers who must use the store. Fix: Publish as PWA to Chrome Web Store.

Net assessment: Deploying the current plan as-is would result in pages that Google sees but won't rank — because the meta is blank (Bug #1), the content is empty stubs (Bug #4), and the fake review schema risks a manual action penalty (Bug #2). The technical intent is correct; the implementation has holes.

Section 04 — Technical Fixes
The Correct Implementation
The exact code changes needed to fix the four critical bugs. These must be done before anything else ships.
Fix #1 — Replace SEOMeta with react-helmet-async (SSG-compatible)
src/components/seo/SEOMeta.tsx — REPLACE WITH:
import { Helmet } from 'react-helmet-async';
import { SITE_CONFIG } from '../../data/seo-config';

export function SEOMeta({ title, description, keywords = [], canonical, structuredData }) {
  const resolvedCanonical = canonical
    ? `${SITE_CONFIG.url}${canonical}`
    : SITE_CONFIG.url;

  return (
    <Helmet>
      <title>{title}</title>
      <meta name="description" content={description} />
      <link rel="canonical" href={resolvedCanonical} />
      <meta property="og:title" content={title} />
      <meta property="og:description" content={description} />
      {structuredData && (
        <script type="application/ld+json">
          {JSON.stringify(structuredData)}
        </script>
      )}
    </Helmet>
  );
}
// Also wrap App.tsx with <HelmetProvider> and configure vite-ssg
// to use renderToStaticMarkup with Helmet extraction

import { ImageResponse } from '@vercel/og';

export const runtime = 'edge';

export default function handler(req) {
  const { searchParams } = new URL(req.url);
  const title = searchParams.get('title') || 'Draw in the Air';
  const emoji = searchParams.get('emoji') || '🌀';

  return new ImageResponse(

    </div>,
    { width: 1200, height: 630 }

}

  • Submit to EdSurge, Edutopia, TES — one-time effort

A /free-resources page with free printable PDF worksheets: "Trace Letter A Worksheet," "Number Tracing 1-10," etc. Teachers who find the PDFs via Google link back from their blogs, TpT stores, and school sites. This is the single most reliable passive link-building strategy for ed-tech.
  • 1 PDF per letter (26 free downloads)
  • 1 PDF per number set (0–10)
  • Each download page links back to the web app
  • Teachers post PDFs + link on classroom blogs
  • Pinterest pins from teachers drive traffic too

YouTube is the #2 search engine. Parents search "trace letter A for kids" on YouTube just as often as Google. A 60-second screen recording of the app, uploaded per activity, creates a second discovery channel and embeds on pages as a trust signal for Google.
  • 1 demo video per major activity (4–6 videos)

  • YouTube backlink from Google's own platform = strong authority

Parents discover kids' activities on Pinterest 40% of the time before Googling. Each activity page with a pin-optimized image gets discovered through Pinterest's algorithm, driving warm, high-intent traffic. Pinterest pins stay indexed for 3–5 years.

Engine 7 — Chrome Web Store + Common Sense Media Listings
These are free distribution channels with massive built-in audiences that are already searching for exactly this product. Neither requires ongoing work after initial listing setup.
